# v1 채점 방식 (원본)

**채점 실행일**: 2026-05-19  
**채점 소요 시간**: 09:24 ~ 10:22 (약 58분)  
**채점 모델**: gpt-4o-mini  
**데이터 생성 모델**: gpt-5.4-mini-2026-03-17  
**대상 파일**: 4500개 (3모델 × 6기법 × 5구간 × 50샘플)

---

## 채점 방식

7개 필드 전체를 GPT 1회 호출로 단일 점수(0.0~1.0) 산출.

```
프롬프트:
"다음 두 JSON을 비교하여 정보 추출 정확도를 0.0~1.0 사이 숫자 하나만 출력하세요.
기준: 1.0=완벽 일치, 0.5=핵심 정보 절반 정도 일치, 0.0=완전 불일치

정답:
{answer}

모델 출력:
{parsed}

점수(숫자만):"
```

max_tokens=5, temperature=0.0으로 고정.  
파싱 실패(is_valid_json=False) 시 semantic_score=0.0으로 고정.

## 실제 불일치 패턴 분석

채점 이후 v2 설계를 위해 4500개 파일에서 필드별 불일치 케이스를 직접 샘플링했다.  
아래 패턴들이 v1 채점에서 어떻게 처리됐는지 알 수 없었다는 것이 핵심 한계다.

### 기관명

모델이 기관명만 출력하고 부서명을 누락하는 패턴이 주로 1.5b에서 발생.

```
정답: 행정안전부 청사시설과
출력: 행정안전부

정답: 경기도교육청 학교시설과
출력: 경기도교육청
```

### 문서번호

빈 문자열, 필드 설명 텍스트 출력, 한자 혼용 등 명백한 오류 패턴.

```
정답: 강남교통-2024-1187
출력: 문서 고유 번호       ← xml 기법에서 발생

정답: 경기도교육청-학생교통-2024-0874
출력: 경기도教育廳-學生交通-2024-0874  ← 한자 혼용
```

### 작성일 / 처리기한

날짜 값은 맞으나 포맷이 다른 경우가 대부분. cot 기법에서 집중적으로 발생.

```
정답: 2024-05-18
출력: 2024년 5월 18일

정답: 2024-06-07
출력: 2024년 6월 7일
```

v1에서는 이 케이스가 내용 불일치로 처리됐을 가능성이 높다.

### 담당자

직책 누락 패턴이 일관되게 나타남. cot 기법에서 집중 발생.

```
정답: 이서연 주무관
출력: 이서연

정답: 박지현 주무관
출력: 박지현
```

### 제목

끝 단어 절삭, 한자 혼용, 단어 누락 등 다양한 패턴.

```
정답: 교육복지 지원사업 운영 규정집(안) 안내
출력: 교육복지 지원사업 운영 규정집(안)     ← 끝 단어 절삭

정답: 교육공무직 인사규정 일부개정안 행정예고
출력: 교육공무직 인사규정 일부개정안 행政예고  ← 한자 혼용
```

## 한계 및 v2 전환 이유

### 1. 날짜 포맷 오채점

모든 프롬프트가 `YYYY-MM-DD` 포맷을 명시적으로 요구했지만, 모델이 `2024년 5월 18일` 형식으로 출력한 경우 GPT가 이를 내용 불일치로 처리했을 가능성이 높다. 날짜 값 자체는 맞는데 감점이 발생한 것이다.

### 2. 필드 뭉개기

7개 필드를 GPT가 단일 점수로 평가했기 때문에 어느 필드가 실패했는지 알 수 없었다. 기관명 부서명 누락, 담당자 직책 누락, 제목 절삭이 모두 같은 점수 범위에 묻혔다. "xml 기법에서 어느 필드가 특히 불안정한가"를 구분할 수 없었다.

### 3. GPT rubric 모호

"핵심 정보 절반 정도 일치 = 0.5" 기준이 불명확해 GPT 재량이 컸다. 실제로 6/7 필드를 완벽히 맞춰도 0.5점이 나오는 케이스가 확인됐다.

### 4. semantic_score 변별력 부재

모델 간, 기법 간 semantic_score 차이가 모두 0.03 수준이었다. 실제 추출 품질 차이가 점수에 반영되지 않고 있어 연구 결과로서의 신뢰도가 낮다.

### 5. 200 구간 이상치 원인 불명

token_budget=200에서 semantic_score가 0.728로 튀는 현상을 v1 채점 방식으로는 설명할 수 없었다. 짧은 문서 효과인지 GPT 채점 편향인지 구분 불가능했다.
